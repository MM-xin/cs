## 第 1 步：库存管理（MVP 核心）

目标：先能管理“我当前持有的饰品”，同时为后续接入 SteamDT 行情、盈亏统计打好数据基础。

### 1.1 数据库接入

- [ ] 引入 SQLite
- [ ] 建立 `items` 表
- [ ] 实现数据库初始化（项目启动自动建表）
- [ ] 为 `name`、`status` 字段创建索引（加速搜索和筛选）
- [ ] 预留 SteamDT 关联字段（`steamdt_id`），后续用于自动更新价格

`items` 表字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | INTEGER, PK | 主键 |
| `name` | TEXT, index | 饰品名称 |
| `category` | TEXT | 类型（步枪/手枪/刀/手套/箱子/印花 等） |
| `wear` | TEXT | 磨损档位枚举：崭新出厂 / 略有磨损 / 久经沙场 / 破损不堪 / 战痕累累 / 无磨损（箱子/印花等） |
| `steamdt_id` | TEXT, nullable | SteamDT 平台的商品 ID，后续用于价格同步 |
| `image_url` | TEXT | 饰品图片地址（初期用占位图，后续可从 SteamDT 回填） |
| `buy_price` | REAL | 买入单价 |
| `sell_price` | REAL, nullable | 卖出单价（未卖出时为空） |
| `current_price` | REAL | 当前估值（初期 = `buy_price`，后续由 SteamDT 更新） |
| `fee_rate` | REAL | 手续费率，默认 `0.01`（1%） |
| `fee_amount` | REAL | 手续费金额（默认按 `sell_price * fee_rate` 计算，允许手动覆盖） |
| `profit` | REAL | 利润（存储计算结果，也可在查询时即时算） |
| `profit_rate` | REAL | 利润率（= `profit / buy_price`） |
| `buy_platform` | TEXT | 买入平台 |
| `buy_time` | DATETIME | 买入时间（可手动修改，用于计算冷却期） |
| `tradable_at` | DATETIME | 可交易时间（根据 `buy_time` 自动计算，见下方规则） |
| `is_tradable` | INTEGER | 当前是否可售：`1` 可售，`0` 冷却中 |
| `status` | TEXT, index | 状态枚举：`in_stock`(在库) / `sold`(已售) / `withdrawn`(撤回) |
| `note` | TEXT | 备注 |
| `created_at` | DATETIME | 记录创建时间（系统自动写入，不可修改） |
| `updated_at` | DATETIME | 更新时间 |

字段说明补充：

- **磨损 `wear`**：使用档位枚举，不录具体浮点值，箱子/印花/挂件等没有磨损的统一填“无磨损”。
- **可售冷却 `tradable_at` / `is_tradable`**：
  - 规则：从 `buy_time` 开始，先取“下一个 16:00”，再加 7 天，得到冷却结束时间
  - 当当前时间 >= `tradable_at` 时，`is_tradable = 1`
  - 因为你可能延迟录入，所以 `buy_time` 必须允许手动修改；`created_at` 只记录你什么时候把这条数据录进系统
- **利润计算规则**：
  - `in_stock`：`profit = current_price - buy_price`，`profit_rate = profit / buy_price`
  - `sold`：`profit = sell_price - buy_price - fee_amount`，`profit_rate = profit / buy_price`
  - `withdrawn`（买家撤回）：`profit = 平台返还的卖家保证金`，具体金额记录到 `sell_price` 字段（表示实际到账），或单独用 `note` 记录；利润率照常用 `profit / buy_price` 计算
- **手续费 `fee_rate` / `fee_amount`**：默认 1%，卖出时自动算好，允许在编辑时手动改。
- **`current_price`**：初期新建饰品时自动 = `buy_price`，后续由 SteamDT 行情同步覆盖。

验收：

- 重启服务后数据不丢失
- 可以在 DB 中看到 `items` 表，且 `name`、`status` 有索引
- `steamdt_id`、`image_url`、`fee_rate`、`sell_price`、`profit`、`profit_rate` 字段齐全

### 1.6 状态流转（含撤回）

状态枚举：`in_stock`（在库） / `sold`（已售） / `withdrawn`（撤回）

- [ ] 在库 → 已售：录入卖出价，自动算手续费与利润
- [ ] 在库/已售 → 撤回：买家撤回后，平台退回的卖家保证金记为该饰品的收益
  - 撤回时输入“实际到账金额”，写入 `sell_price`
  - `profit = sell_price - buy_price`（撤回一般不再额外扣手续费，如有则减去）
  - 状态标记为 `withdrawn`
- [ ] 撤回状态可重新改回“在库”（如果饰品又回到库存）

验收：

- 三种状态之间切换正确，利润字段跟随更新
- 撤回场景能正确记录“退回的保证金”并体现在利润中
